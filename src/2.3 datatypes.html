Типы данных

Значение в JavaScript всегда имеет определённый тип. Например, строка или число. В JavaScript существует восемь основных типов данных. Здесь мы рассмотрим их в общих чертах, а в следующих главах поговорим о каждом из них подробнее.

Мы можем поместить любой тип в переменную. Например, переменная может в какой-то момент быть строкой, а затем хранить число:

123 
// no error
let massage  ="hello";
massage = 12345;

Языки программирования, которые допускают такие вещи, такие как JavaScript, называются “динамически типизированными”, что означает, что существуют типы данных, но переменные не привязаны ни к одному из них.

Числа

Тип number представляет как целые числа, так и числа с плавающей запятой.

12 
let message = 123;
n = 12.345

Существует множество операций с числами, например, умножение *, деление /, сложение +, вычитание - и так далее.

Помимо обычных чисел, существуют так называемые “специальные числовые значения”, которые также принадлежат этому типу данных: Infinity, -Infinity и NaN.

1. Infinityпредставляет математическую бесконечность ∞. Это специальное значение, которое больше любого числа. Мы можем получить это в результате деления на ноль:

1
alert(1/0); /Infinity

Или просто ссылайтесь на него напрямую:

1
alert(Infinity); /Infinity

2. NaNпредставляет вычислительную ошибку. Это результат неправильной или неопределенной математической операции, например:

1
alert("not a number"/2); /NaN

Строка

Тип string используется для представления текстовых данных. Строки должны быть заключены в кавычки: одинарные, двойные или обратные (для шаблонных строк).

123 
let str = "Привет!";
let str2 = 'Одиночные кавычки тоже подойдут';
let phrase = `Обратные кавычки позволяют встраивать переменные ${str}`;

Двойные и одинарные кавычки - это “простые” кавычки. В JavaScript между ними практически нет разницы.

Шаблонные строки, заключённые в обратные кавычки, могут содержать выражения, которые будут вычислены и вставлены в строку, заключая их в ${…}:

12 
name = "John"; 
alert = `Привет,${name}`; // Привет, John!

Пожалуйста, обратите внимание, что это можно сделать только с помощью обратных ссылок. Другие кавычки не имеют такой функции встраивания!

Не существует символьного типа

В некоторых языках существует специальный тип “character” для отдельного символа. Например, в языке C и Java он называется “char".

В JavaScript такого типа нет. Есть только один тип: string. Строка может состоять из нуля символов (быть пустой), одного символа или нескольких из них.

Boolean (логический тип)

Тип boolean имеет всего два значения: true и false.

Этот тип обычно используется для хранения значений “да”/ "нет": true означает "да, правильно", а false означает “нет, неверно”.

123 
let nameFieldChecked = true; // Да, поле заполнено

let ageFieldChecked = false; // Нет, поле не заполнено

Булевы значения часто используются в условных операциях:

123 
const age = 18;
let isAdult = age < 12;
alert(isAdult); //false

Значение “null”

Специальное значение null представляет собой "ничего", "пусто" или "значение неизвестно".

1 
let age = null;

null не равен ни одной другой переменной и используется, чтобы явно указать на отсутствие значения.

В приведенном выше коде указано, что age неизвестно.

Значение “undefined”

Тип undefined означает, что переменная была объявлена, но ещё не присвоено значение.

123 
let age;

alert(age); //показывает "undefined"

Технически возможно явное присвоение undefined переменной:

123456 
let age = 52;

// изменяем значение на undefined
age = undefined;
alert (age); // "undefined"

Так делать не рекомендуется. Обычно null используется для присвоения переменной «пустого» или «неизвестного» значения, а undefined – для проверок, была ли переменная назначена.

Объекты и символы

Тип object (объект) – особенный.

Все остальные типы называются «примитивными», потому что их значениями могут быть только простые значения (будь то строка, или число, или что-то ещё). В объектах же хранят коллекции данных или более сложные структуры. Объекты занимают важное место в языке и требуют особого внимания.

Тип symbol (символ) используется для создания уникальных идентификаторов в объектах. Мы упоминаем здесь о нём для полноты картины, изучим этот тип после объектов.

BigInt

Тип BigInt представляет целые числа произвольной длины. Чтобы создать значение типа BigInt , достаточно добавить n в конец числового литерала.

1
const bigInt =14619540n;

BigInt позволяют безопасно работать с целыми числами, превышающими диапазон, поддерживаемый типом number.

Оператор typeof

Оператор typeof возвращает тип аргумента. Это полезно, когда мы хотим обрабатывать значения различных типов по-разному или просто хотим сделать проверку.

У него есть две синтаксические формы:

1234 
// Обычный синтаксис
typeof 52 // Выведет "number"
// Синтаксис, напоминающий вызов функции (встречается реже)
typeof(52)// Также выведет "number"

Если передается выражение, то нужно заключать его в скобки, т.к. typeof имеет более высокий приоритет, чем бинарные операторы:

123 
typeof 25 + "Машин"; // Выведет "number Машин"

typeof (25 + "Машин");// Выведет "string"

Другими словами, скобки необходимы для определения типа значения, которое получилось в результате выполнения выражения в них.

Пример использования всех типов

12345678910 
let name = "Alice"; // Строка
let age = 25; // Число
let isStudent = true; // Булевый
let height = null; // null
let weight; // undefined
let id = Symbol("id"); // Символ
const bigNumber=14619540n; // bigInt
let Math // "object"
alert(...) // "function"

Эти основные типы данных и операции с ними являются фундаментальными концепциями JavaScript, которые необходимо понять для эффективного программирования на этом языке.
