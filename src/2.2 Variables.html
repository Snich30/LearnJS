Переменные

Что же такое переменная и что она из себя представляет?

Переменные в JavaScript - это контейнеры, которые содержат значения данных. Они используются для хранения информации, такой как числа, строки, булевы значения, объекты и многое другое. При работе с переменными важно понимать их типы данных, область видимости и правила именования.

Давайте рассмотрим это подробнее, начиная с объявления переменных.

Обьявление переменных

В JavaScript переменные могут быть объявлены с использованием ключевых слов let , var и const.

1. var : Это старый способ объявления переменных. Переменные, объявленные с помощью var , имеют функциональную область видимости.

1
var message;

2. let : Появился в ECMAScript 6 (ES6). Переменные, объявленные с помощью let , имеют блочную область видимости.

1
let message;

3. const : Также появился в ECMAScript 6. Переменные, объявленные с помощью const, имеют блочную область видимости, и их значения нельзя изменить после присвоения.

1
const message;

Работа с переменной let

Есть тонкие различия между let и var , переменная var - почти тоже самое, что и переменная let , но суть в том, что оно объявляет переменную, но немного по-другому, «устаревшим» способом.

Теперь можно поместить в переменную let данные (другими словами, определить переменную), используя оператор присваивания =:

123 
let message;

message = 'Привет мир!';

Строка сохраняется в области памяти, связанной с переменной. Мы можем получить к ней доступ, используя имя переменной:

1234 
let message;
message = 'Привет мир!'; 
              
alert(massage);//показывает содержимое переменной

Для краткости можно совместить объявление переменной и запись данных в одну строку:

123 
let message = 'Привет мир!';

alert(massage);//Привет мир

Мы также можем объявить несколько переменных в одной строке:

1
let user = 'John', age = 23, massege = 'John';

Такой способ может показаться короче, но мы не рекомендуем его. Для лучшей читаемости объявляйте каждую переменную на новой строке.

Многострочный вариант немного длиннее, но легче для чтения:

123 
let user = 'John';
let age = 25;
let message = 'Hello';

Можно записать короче этот код:

123 
let user = 'John';
  , age = 25;
  , message = 'Hello';

Правила именования переменных

1. Имена переменных могут содержать буквы, цифры, знак подчеркивания (_) и знак доллара ($). Они не могут начинаться с цифры.

2. Имена переменных чувствительны к регистру (age, Age и AGE - разные переменные).

3. Имена переменных не должны совпадать с зарезервированными словами JavaScript, такими как function, let, const, и т.д.

Примеры допустимых имён:

12 
let userName;
let numb123;

Если имя содержит несколько слов, обычно используется верблюжья нотация, то есть, слова следуют одно за другим, где каждое следующее слово начинается с заглавной буквы: myVeryLongName.

Самое интересное – знак доллара '$' и подчёркивание '_' также можно использовать в названиях. Это обычные символы, как и буквы, без какого-либо особого значения.

Эти имена являются допустимыми:

1234 
let $ = 2; // объявили переменную с именем "$" 
let _ = 4; // объявили переменную с именем "_" 
              
alert($ + _);// 6

Примеры неправильных имён переменных:

123 
let 2a; // не может начинаться с цифры

let my-name;// дефис '-' не разрешён в имени

Нелатинские буквы разрешены, но не рекомендуются

Можно использовать любой язык, включая кириллицу или даже иероглифы, например:

12 
let имя = '...';
let 我 = '...';

Технически здесь нет ошибки, такие имена разрешены, но есть международная традиция использовать английский язык в именах переменных. Даже если мы пишем небольшой скрипт, у него может быть долгая жизнь впереди. Людям из других стран, возможно, придётся прочесть его не один раз.

Константы

Чтобы объявить константную, то есть, неизменяемую переменную, используйте const const вместо let:

1
const myBirthday ='01.04.2003';

Переменные, объявленные с помощью const, называются «константами». Их нельзя изменить. Попытка сделать это приведёт к ошибке:

123 
const myBirthday = '01.04.2003';

myBirthday = '11.01.2005'; // ошибка, константу нельзя перезаписать!

Eсли программист уверен, что переменная никогда не будет меняться, он может гарантировать это и наглядно донести до каждого, объявив её через const.

Придумывайте правильные имена

Правильные имена переменных - ключ к пониманию вашего кода и улучшению его читаемости. Придумывая имена для переменных, функций и других элементов вашего кода, следует придерживаться определенных правил и лучших практик. В этой секции мы рассмотрим, как выбирать подходящие имена и почему это важно для качественной разработки.

Зачем важны правильные имена

1. Читаемость кода: Понятные имена переменных делают ваш код более читаемым для других разработчиков (и для вас самого в будущем). Это упрощает процесс отладки и сопровождения кода.

2. Ясность намерений: Имена переменных должны отражать их назначение и намерения. Чем более ясно и точно они описывают предназначение переменной, тем проще понять ее использование в коде.

3. Избегание ошибок: Правильные имена могут помочь избежать ошибок в коде. Например, если вы правильно назовете переменную, описывающую текущий пользовательский ввод, будет проще избежать путаницы при работе с этим значением.

Лучшие практики при выборе имен

• Описательность: Используйте имена, которые ясно описывают предназначение переменной или элемента кода.

• Используйте camelCase: Для имен переменных в JavaScript принято использовать camelCase (начиная с маленькой буквы, каждое новое слово с заглавной).

• Избегайте сокращений: Полные имена переменных предпочтительнее сокращений, если только сокращение не широко распространено и понятно всем разработчикам.

• Используйте английский язык: Для имен переменных используйте английский язык, чтобы ваш код был более доступен для международного сообщества разработчиков.

Примеры:

1.Плохо:

12 
let a = 10; // что представляет собой "a"?
let usrNm = 'John'; // cлишком сокращенное имя переменной

2.Хорошо:

12 
let itemCount = 10; // ясное описание переменной
let userName = 'John'; // описательное имя переменной.

На заметку:

Выбирая правильные имена для ваших переменных и других элементов кода, вы сделаете свой код более понятным и удобным для работы. Помните, что хорошие имена - это инвестиция в читаемость и поддерживаемость вашего кода.
